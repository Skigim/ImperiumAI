Tier 2: The How

These roles protect the kernel, memory model, and CPU budget while implementing the approved gameplay semantics.

Engineering Core

Lead Architect

  • Maps specs onto Imperium layers
  • Classifies CPU risk and affected systems
  • Breaks ties using the priority hierarchy
Engineering Core

State & Memory Engineer

  • Owns interfaces, memory schema, and cache shape
  • Defines lifecycle, cleanup, and serialization rules
  • Prevents Memory bloat and contract drift
Engineering Core

Implementation Dev

  • Writes bounded per-tick TypeScript logic
  • Reuses shared primitives instead of duplicating systems
  • Optimizes without silently changing gameplay semantics

Global Priority Hierarchy

1. Kernel SafetyMain loop, scheduler stability, bounded execution.
2. Emergency DefenseImmediate hostile response and catastrophic protection.
3. Room SurvivalDowngrade prevention, energy continuity, minimum colony function.
4. State SafetyMemory integrity, schema correctness, cleanup guarantees.
5. Economy ContinuityIncome, hauling, spawn service, reserve stability.
6. Expansion ProgressClaims, reserves, remotes, and bootstrap sequencing.
7. Industry OptimizationLabs, terminals, boosts, factory, market.
8. Opportunistic OffenseDiscretionary combat and nonessential aggression.

Fast-track lane: Tier 1 may update existing tuning surfaces directly, but the change still requires an automated TypeScript compile or lint pass before it is considered safe.
